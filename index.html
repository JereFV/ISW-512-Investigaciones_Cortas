Usabilidad Web

  • Definición
  • Importancia
  • Principios

¿Que es?

La usabilidad web se refiere a aquellas características y atributos que determina que tan fácil es de usar un sitio web para los visitantes. Se enfoca en aspectos como el diseño gráfico, presentación y organización de la información, y en el funcionamiento mismo.

El objetivo principal es que el usuario navegue con una mínima inversión de tiempo y esfuerzo. Este proceso empieza desde que el usuario entra al sitio y toma en consideración cada uno de los elementos que lo componen y con los que tiene interacción.

Importancia

La falta de usabilidad web en una página es la mayor causa de abandono de estas. Es decir, si en nuestro sitio web la información es difícil de encontrar, tiene pantallas de carga demasiado lentas o no tiene un esquema claro de navegación, probablemente los internautas cierren la pestaña sin siquiera leer el primer parráfo completo. Es fundamental recordar, que si perdemos a un usuario, estamos perdiendo un potencial cliente.

Al darle importancia a la experiencia de usuario y en general a la usabilidad de un sitio web, las empresas pueden atraer nuevos usuarios, mantener los actuales y establecer su marca comercial de manera on-line. Algunos beneficios de una correcta usabilidad web:

  • Logra conseguir más tráfico
  • Aumenta la duración de las visitas en la página.
  • Mejora la experiencia y satisfacción de los visitantes.

Principios

Los principios de usabilidad web de Jakob Nielsen son la base de cualquier página web para que sea “user friendly”. O lo que es lo mismo, es un tipo de diseño centrado en el usuario para conseguir mejorar la experiencia del mismo.

La tabla inferior describe cada uno de los 10 principios de usabilidad web de Jakob Nielsen:

Principio Definición Ejemplos
1. Visibilidad del estado del sistema Este principio de usabilidad web nos indica que siempre tenemos que tener informado al usuario de lo que está pasando en nuestra web y ofrecerle una respuesta en el menor tiempo posible.
  • Las barras de carga de los procesos de descarga de archivos.
  • Los indicadores de los procesos de compra.
2. Relación entre el sistema y el mundo real El sitio web o aplicación tiene que utilizar el lenguaje del usuario, con expresiones y palabras que le resulten familiares. Además la información debe aparecer en un orden lógico y natural.
  • El símbolo de papelera utilizado para aquellas acciones de borrado de información.
3. Control y libertad del usuario En caso de elegir alguna opción del sitio web por error, el usuario agradecerá disponer de una “salida de emergencia” para abandonar el estado no deseado en que se halla. Debe poder deshacer o repetir una acción previamente realizada.
  • Botón de deshacer
  • Dar la posibilidad de editar un perfil personal.
4. Consistencia y estándares Es importante establecer convenciones lógicas y mantenerlas siempre. El usuario no tiene por qué saber que diferentes palabras, situaciones o acciones significan lo mismo.
  • Icono de barras horizontales para un menú desplegable.
  • El logo de la página en el encabezado alineado a la izquierda
5. Prevención de errores Tenemos, en todo lo posible, que prevenir cualquier error que pueda cometer el usuario. Y dado el caso de que este cometa uno, tenemos que poner a su alcance todas las opciones posibles para poder corregirlo."a
  • Confirmación de dirección de correo electrónico o de la contraseña.
  • Comprobación de campos requeridos en formularios en tiempo real.
6. Reconocimiento antes que recuerdo Este principio dicta que debemos hacer visibles acciones y opciones para que el usuario no tenga que recordar información entre distintas secciones o partes del sitio web o aplicación.
  • La previsualización de las fuentes en los editores de texto.
  • Los enlaces auxiliares de una home que incluyen texto más icono para reforzar el mensaje.
7. Flexibilidad y eficiencia de uso Tenemos que tener un sitio web preparado para todo tipo de usuario, desde los más novatos hasta los más experimentados. Si conseguimos que cualquiera pueda navegar por nuestra web logramos flexibilidad. Y si tenemos opciones para los más experimentados obtenemos eficiencia.
  • La búsqueda de Google avanzada con operadores.
  • Atajos de teclado para ciertas funcionalidades.
8. Diseño estético y minimalista Las páginas web no deben contener información innecesaria, evita lo máximo posible el recargamiento del sitio ya sea con contenido o elementos visuales así como funcionalidades que toman varios segundos en su procesamiento, el usuario busca páginas limpias y que carguen rápido.
  • La tendencia actual minimalista de los sitios web de grandes compañias como Apple o PlayStation.
9. Ayudar a los usuarios a reconocer, diagnosticar y corregir los errores Los mensajes de error se deben entregar en un lenguaje claro y simple, indicando en forma precisa el problema y sugerir una solución constructiva al problema.
  • La manipulación de códigos de error devuelto por algún servicio web en un formato legible al usuario.
  • Formularios que avisan en tiempo real los errores que cometes al rellenarlo.
10. Ayuda y documentación Siempre tenemos que darle al usuario la posibilidad de tener un pequeño manual de funcionamiento, previniendo cualquier situación donde se amerite. Esta ayuda debe ser fácil de localizar, definir los pasos claramente y no ser muy extensa.
  • FAQs, preguntas frecuentes.
  • El icono de interrogación cerca de algunas opciones.
